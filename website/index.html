The problem

Every domain you look up is a record someone keeps.

Your router asks an upstream resolver to translate every hostname your devices reach. That resolver, along with your ISP, can tie each question back to your household address and build a profile over time. Encrypting the query hides its contents, not its origin.

Torhole breaks the link between the question and you. The resolver still answers, but it sees a Tor exit, not your public IP. Filtering and encryption come along for the ride.

What happens to a lookup

Four hops. One narrow, testable promise.

01Pi-holeFilters unwanted domains and answers local DNS under your control.
02dnscrypt-proxyEncrypts the upstream resolver connection using DNSCrypt or DNS-over-HTTPS.
03Tor circuitSeparates the request from your public IP before it reaches the resolver.
04ResolverReceives the query from a Tor exit address, then returns the answer.

// Ordinary web, video, and app traffic keeps its normal route. Only DNS takes this path.

Honest scope

Torhole is not a VPN.

It deliberately protects the DNS path and nothing wider. Naming what it doesn't do is how you can trust what it does.

  • Does not route web, video, or app traffic through Tor.
  • Does not replace your public IP when you visit a site.
  • Does not make browsing anonymous or secure an unsafe device.
  • Cannot protect apps that ignore your DNS and use their own resolver.

Proof, not a promise

The dashboard shows evidence, not a checkmark.

Torhole records the observed Tor exit, asks the Tor Project whether that address is a relay, and keeps the result. Unlabelled circuits stay unlabelled. The interface doesn't invent certainty.

Install

One command on a clean Debian or Ubuntu host.

The bootstrap downloads Torhole, asks once for host authorization, and opens a private setup URL where you choose Home or Advanced and watch it deploy. Raspberry Pi 5 is the reference small-device platform.

torhole-host / bash
# download, verify, and launch the guided installer
$curl -fsSL https://raw.githubusercontent.com/torhole/torhole/main/get-torhole.sh | bash

Prefer to read it first? curl -fsSLO …/get-torhole.sh then less get-torhole.sh. The traditional git clone → ./install.sh flow starts the same installer.
